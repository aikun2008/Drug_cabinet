# 药品柜管理系统小程序 - 学生端

## 项目概述
基于微信小程序的药品柜智能管理系统学生端，实现药品信息查询、使用记录管理、个人中心等功能。

## 核心功能说明

### 1. 用户认证
- **功能**：实现用户名密码登录，管理登录状态
- **文件**：`app.js` (全局状态管理)、`login/login.js` (登录逻辑)
- **流程**：登录 → 获取token → 存储token → 更新全局状态 → 页面跳转
- **技术点**：token验证、本地存储、状态管理

### 2. 药品管理
- **功能**：药品信息查询、详情查看
- **文件**：`drugInfo/drugInfo.js` (药品逻辑)
- **流程**：搜索药品 → 查看列表 → 点击查看详情
- **技术点**：API调用、搜索功能、列表渲染、模态框交互

### 3. 学生核心功能
- **功能**：学生常用功能入口
- **文件**：`index/index.js` (首页逻辑)
- **功能点**：
  - 🔍 查询药品：跳转到药品信息页面
  - 📋 申请药品：提交药品申请
  - 📤 借药：办理药品借出手续
  - 📥 还药：办理药品归还手续
- **技术点**：功能入口、弹窗交互、API调用

### 4. 使用记录
- **功能**：药品使用记录查询与筛选
- **文件**：`record/record.js` (记录逻辑)
- **筛选**：全部、借出、归还三种类型
- **技术点**：数据筛选、列表渲染、状态显示

### 5. 个人中心
- **功能**：用户信息管理和系统设置
- **文件**：`profile/profile.js` (个人中心逻辑)
- **功能点**：修改密码、关于系统、退出登录
- **技术点**：用户信息展示、退出登录逻辑、本地存储清除

## 开发与运行

### 1. 开发环境
- **前端**：微信开发者工具
- **后端**：Python 3.7+, Flask, MySQL, Redis
- **网络**：局域网连接（本地开发）
- **账号**：微信小程序账号（测试号或正式号）

### 2. 运行步骤

#### 步骤1：启动后端服务
1. 进入后端项目目录：`cd Drug_cabinet_server`
2. 激活虚拟环境：`venv\Scripts\activate`
3. 安装依赖：`pip install -r requirements.txt`
4. 启动服务：`python main.py`
5. 确认服务运行在 `http://localhost:5000`

#### 步骤2：配置小程序
1. 在微信开发者工具中导入项目：`Drug_cabinet_MP`
2. 打开 `app.js` 文件，修改 `baseUrl` 配置：
   ```javascript
   globalData: {
     isLoggedIn: false,
     userInfo: null,
     baseUrl: 'http://localhost:5000/api'  // 确保与后端服务地址一致
   }
   ```
3. 在微信开发者工具中，点击右上角"详情" → "本地设置" → 勾选"不校验合法域名"

#### 步骤3：运行小程序
1. 点击微信开发者工具中的"编译"按钮
2. 使用用户名密码登录系统
3. 体验各页面功能

### 3. 注意事项
- **后端服务**：确保 `Drug_cabinet_server` 服务正在运行
- **网络连接**：手机和电脑需在同一局域网内
- **域名校验**：开发阶段必须勾选"不校验合法域名"选项
- **API地址**：确保 `app.js` 中的 `baseUrl` 与后端服务地址一致
- **数据库**：确保 MySQL 数据库已启动，且配置正确
- **Redis**：确保 Redis 服务已启动（用于缓存，提高性能）
